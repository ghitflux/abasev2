## Problemas encontrados e soluções

### 1. `jti_hex` NOT NULL sem default (login 500)
**Causa:** Coluna `token_blacklist_outstandingtoken.jti_hex` era `NOT NULL` sem default. `simplejwt 5.4.0` não envia o campo no INSERT.
**Solução:**
```sql
ALTER TABLE token_blacklist_outstandingtoken
  MODIFY jti_hex varchar(255) NOT NULL DEFAULT '';
```

### 2. Timeout do `docker build` via paramiko
**Causa:** Build demora > 10min; paramiko encerra o canal por timeout.
**Solução:** Usar `nohup ... & echo $!` + canal com `settimeout(10)` para capturar o PID imediatamente, depois monitorar via polling do log.

### 3. Conflito de nome de rede (hostinger_abase-net vs abase_abase-net)
**Causa:** O compose file em `deploy/hostinger/` gerava project name `hostinger`, criando a rede `hostinger_abase-net`. Os containers MySQL/Redis usam `abase_abase-net`.
**Solução:** Passar `-p abase` explicitamente no `docker compose` para usar o mesmo project name original.

### 4. `SECRET_KEY` vazio → backend em restart loop
**Causa:** O `docker compose up` estava sendo chamado sem `--env-file`, portanto as variáveis de ambiente ficavam em branco.
**Solução:** Sempre usar `--env-file /opt/ABASE/env/.env.production` em todos os comandos compose.
