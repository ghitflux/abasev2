### 1. Atualizar o código no servidor

```bash
cd /opt/ABASE/repo
git fetch origin
git pull origin abaseprod
```

Resultado esperado:

- chegada do commit deste pacote

### 2. Rebuild das imagens

Como houve alteração em backend, frontend, compose local e configuração de runtime do Next, manter rebuild completo de backend e frontend.

```bash
docker compose -p abase --env-file /opt/ABASE/env/.env.production \
  -f deploy/hostinger/docker-compose.prod.yml \
  build --no-cache backend frontend
```

Se o build estiver demorando demais na sessão remota, usar o mesmo padrão de execução em background documentado no deploy de 29/03/2026:

```bash
nohup docker compose -p abase --env-file /opt/ABASE/env/.env.production \
  -f deploy/hostinger/docker-compose.prod.yml \
  build --no-cache backend frontend \
  > /tmp/abase_build_2026-03-31.log 2>&1 &
```

### 3. Recriar backend e frontend

```bash
docker stop abase-backend-prod abase-frontend-prod
docker rm abase-backend-prod abase-frontend-prod

docker compose -p abase --env-file /opt/ABASE/env/.env.production \
  -f deploy/hostinger/docker-compose.prod.yml \
  up -d --no-deps --force-recreate backend

  -f deploy/hostinger/docker-compose.prod.yml \
```

## Problemas já conhecidos e cuidados

### 1. Sempre usar `-p abase`

Evita repetir o problema de rede divergente entre compose local do diretório `deploy/hostinger/` e a stack principal.

### 2. Sempre usar `--env-file /opt/ABASE/env/.env.production`

Evita containers subindo sem `SECRET_KEY` e outras variáveis obrigatórias.

### 3. Não confiar em `GET` para validar login da API

O endpoint `/api/v1/auth/login/` é de `POST`; um `405` em `GET` indica que a rota está viva.

### 4. Rebuild completo é recomendado neste pacote

Há mudanças em backend e frontend no mesmo conjunto, então não vale tentar deploy parcial.
