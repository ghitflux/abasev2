## Problemas encontrados e correções

### 1. `restore_users.sql` com coluna `username` inexistente

O script de restore gerado na sessão anterior usava a coluna `username`, que não existe no modelo atual (`accounts_user` usa apenas `email` como identificador). Foi necessário gerar `restore_users_v2.sql` usando apenas as colunas corretas.

### 2. Backend em restart loop — migration `token_blacklist.0001_initial`

**Erro:**
```
django.db.utils.OperationalError: (1050, "Table 'token_blacklist_blacklistedtoken' already exists")
  Applying token_blacklist.0001_initial...
```

**Causa:** o dump local continha as tabelas do `token_blacklist` mas não os registros correspondentes em `django_migrations`. O backend tentou criar as tabelas novamente e falhou.

**Correção:**
```sql
INSERT IGNORE INTO django_migrations (app, name, applied) VALUES
  ('token_blacklist', '0001_initial', NOW()),
  ('token_blacklist', '0002_outstandingtoken_jti_hex', NOW()),
  ('token_blacklist', '0003_auto_20171017_2007', NOW()),
  ('token_blacklist', '0004_auto_20171017_2013', NOW()),
  ('token_blacklist', '0005_remove_outstandingtoken_jti', NOW()),
  ('token_blacklist', '0006_auto_20171017_2113', NOW()),
  ('token_blacklist', '0007_auto_20171017_2214', NOW()),
  ('token_blacklist', '0008_migrate_to_bigautofield', NOW()),
  ('token_blacklist', '0010_fix_migrate_to_bigautofield', NOW()),
  ('token_blacklist', '0011_linearizes_history', NOW()),
  ('token_blacklist', '0012_alter_outstandingtoken_user', NOW());
```

**Reaplicar em futuros imports:** qualquer import de dump que não contenha os registros de `django_migrations` do `token_blacklist` exigirá essa correção manual.

### 3. Bug simplejwt 5.4.0 — `jti_hex` sem DEFAULT

Reaplicado o fix já conhecido:

```sql
ALTER TABLE token_blacklist_outstandingtoken
  MODIFY jti_hex varchar(255) NOT NULL DEFAULT '';
```

### 4. Nginx retornando 502 em `/login`

Após subir o frontend, o nginx ainda apontava para o upstream anterior. Resolvido com:

```bash
docker exec abase-nginx-prod nginx -s reload
```

## Observações para futuros imports

- **sempre gerar `restore_users_v2.sql`** sem coluna `username` — o schema atual usa somente `email`
- **sempre inserir manualmente** os registros de migração do `token_blacklist` após import de dump que não os contenha
- **sempre reaplicar** o fix `DEFAULT ''` em `token_blacklist_outstandingtoken.jti_hex`
- **recarregar nginx** após subir o frontend: `docker exec abase-nginx-prod nginx -s reload`
- o build cache pode ser purgado antes de restores grandes para liberar espaço: `docker builder prune -f`
- extração de tar.gz grande: usar montagem direta no volume via `docker run alpine tar -xzf` para evitar intermediário em `/tmp`
